## Status Report Email Template

Use this structure for regular status reports and deliverable updates.

### **Subject Line**

```
[Deliverable Type] for [Time Period]
```

**Examples:** 
- `Weekly Summary for January 20-26, 2026`
- `Project Analysis - Q1 2026 Data Review`
- `Monthly Deliverable for February 2026`

---

### **Email Body Structure**

#### **Header Section**

```
[Deliverable Type] for [TIME PERIOD]

[Optional: Special context note about scope or coverage]
```

**Examples of header notes:**
- "Report includes historical analysis (2020-2025)"
- "Analysis covers [specific scope or limitations]"
- "Deliverable includes [additional context]"

---

#### **Key Changes Summary**

Document what changed from previous period or baseline.

```
**Key Changes**

Increases:
- [Metric/Category]: [Change %/Amount], [Reason/Context]
- [Metric/Category]: [Change %/Amount], [Reason/Context]

Decreases:
- [Metric/Category]: [Change %/Amount], [Reason/Context]

No Changes:
- [Summary of consistent metrics or observations]
```

**Example:**

```
**Key Changes**

Increases:
- Data completeness: +8% (new validation rules implemented)
- Processing volume: +12% (expanded coverage area)

Decreases:
- Error rate: -5% (process improvements applied)

No Changes:
- Core metrics stable and within expected range
- Overall quality consistent with historical trends
```

**Writing Tips:**
- âœ… Quantify changes with percentages or concrete numbers
- âœ… Explain reasons (verified in source? Process change? Expected variance?)
- âœ… Note patterns (matches prior periods? First time occurrence?)
- âœ… Distinguish expected changes from anomalies

#### **Structure Documentation**

List and describe components of your deliverable.

```
**Deliverable Structure**

**Component 1:** [Name]
- Description: [What it contains]
- Coverage: [Scope or boundaries]

**Component 2:** [Name]
- Description: [What it contains]
- Coverage: [Scope or boundaries]
```

**Example:**

```
**Report Structure**

**Executive Summary**
- High-level findings and recommendations
- Key metrics and trends
- Critical action items

**Detailed Analysis**
- Methodology and data sources
- Findings by category
- Supporting evidence and charts

**Appendix**
- Raw data tables
- Technical notes
- References
```

#### **Findings by Category**

Document observations, patterns, and analysis for each area.

```
**Findings by Category**

**[Category 1]:**
- [Finding type]: [Detailed observation and context]
- [Pattern note]: [What was verified or confirmed]
- [Special note]: [Additional relevant information]

**[Category 2]:**
- [Repeat pattern]
```

**Detailed Example:**

```
**Findings by Category**

**Data Quality:**
- Completeness improved: 95% of required fields now populated (up from 87% last quarter). Improvement driven by enhanced validation rules implemented in January.
- Consistency verified: Spot-checked 50 records against source; 100% match rate.
- No anomalies detected in core data elements.

**Volume Analysis:**
- Record count: 15,423 items processed. Consistent with historical trend (+/- 3%).
- Peak processing day: Wednesday (22% of weekly volume). Matches expected pattern.
- No capacity issues encountered.

**Process Performance:**
- Average processing time: 2.3 minutes per item (target: <3 minutes). Performance stable.
- Error rate: 0.8% (target: <1%). Within acceptable range.
- Manual intervention required: 12 cases (0.08%). All documented in notes.
```

**Writing Tips:**
- âœ… Start with category name
- âœ… Document findings with specific evidence
- âœ… Provide context (is this expected or new?)
- âœ… Reference verification methods
- âœ… Note patterns to help others understand behavior
- âœ… Flag unexpected issues clearly

## Documentation Best Practices

### **1. Clarity & Specificity**
- âœ… **DO:** "Processing volume increased 15% this week due to seasonal trend; validated against historical patterns"
- âŒ **DON'T:** "Volume went up"

### **2. Quantify Everything**
- âœ… **DO:** "Identified 8 edge cases out of 12,540 records (0.06%)"
- âŒ **DON'T:** "Found a few issues"

### **3. Verify and Cite Sources**
- âœ… **DO:** "Verified via spot-check of 50 records; 100% match to source data"
- âŒ **DON'T:** "Looks correct"

### **4. Document Patterns**
- âœ… **DO:** "This matches Q1 2025 pattern; consistent seasonal behavior"
- âŒ **DON'T:** "It's normal" (without context)

### **5. Flag Issues Proactively**
- âœ… **DO:** "IN PROGRESS - Awaiting clarification on data field X; expected resolution by Feb 20"
- âŒ **DON'T:** "There's a problem" (too vague)

### **6. Maintain Consistency**
- âœ… **DO:** Use same structure each period; makes trends visible
- âŒ **DON'T:** Change format arbitrarily

### **7. Provide Context**
- âœ… **DO:** "Error rate 0.8% (target: <1%, historical average: 1.2%)"
- âŒ **DON'T:** "Error rate 0.8%" (without benchmarks)

### **8. Use Professional Tone**
- âœ… **DO:** "Analysis complete; deliverable ready for review"
- âŒ **DON'T:** "Done!" or overly casual language

```markdown

[Key points discussed]

## Action Items

| Action | Owner | Due Date | Status |
|--------|-------|----------|--------|
| [Task] | [Name] | [Date] | [Not Started/In Progress/Complete] |

## Follow-Up

[Next steps, future meeting dates, open questions]
```

## File Naming Conventions

### **For Reports**
```
[DocType]_[Topic]_[YYYYMMDD].md
```

**Examples:**
- `WeeklyReport_DataAnalysis_20260216.md`
- `ProjectSummary_CustomerRetention_20260131.md`

### **For Meeting Notes**
```
MeetingNotes_[Topic]_[YYYYMMDD].md
```

**Example:** `MeetingNotes_Q1Planning_20260210.md`

### **For Documentation**
```
[ProcessName]_Guide.md
[ToolName]_Documentation.md
```

**Examples:**
- `DataValidation_Guide.md`
- `Dashboard_Documentation.md`

| Situation | Use This Format |
|-----------|-----------------|
| Weekly status update | Status Report Email Template |

## Quick Reference Checklist

Before sending any communication:

- [ ] Subject line clear and specific
- [ ] Key changes quantified with context
- [ ] Findings documented with evidence
- [ ] Verification steps listed
- [ ] Status clearly stated
- [ ] Next steps identified with owners
- [ ] Professional tone maintained
- [ ] Proofread for clarity and accuracy
- [ ] Relevant files attached or linked
- [ ] Recipients appropriate for content
